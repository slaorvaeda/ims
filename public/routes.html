1. Product Master (products)

  • id bigint (PK, Auto-increment)
  • product_id varchar (Unique Code, e.g. A, B)
  • product_name varchar
  • sku varchar (Unique index)
  • fsn varchar (Nullable, Flipkart ID)
  • asin varchar (Nullable, Amazon ID)
  • updated_by varchar (Nullable, Editor User)
  • created_at / updated_at datetime

2. Purchase Master (purchases)

  • id bigint (PK)
  • product_id bigint (FK -> products.id)
  • date date (Purchase date)
  • vendor_id varchar (Vendor Code)
  • quantity int (Purchased quantity)
  • price decimal (10,2)
  • amount decimal (10,2) (Price * Qty)
  • updated_by varchar
  • created_at / updated_at datetime

3. Inward ItemCode Master (inward_item_codes)

  • id bigint (PK)
  • product_id bigint (FK -> products.id)
  • uid varchar (Unique serial e.g. Zig0001)
  • quantity int (Default 1)
  • status varchar (Good Inventory / Damaged)
  • updated_by varchar
  • created_at / updated_at datetime

4. SaleMaster (sales)

  • id bigint (PK)
  • portal_id varchar (Amazon / Flipkart)
  • product_id bigint (FK -> products.id)
  • order_date date
  • quantity int (Sold count)
  • updated_by varchar
  • created_at / updated_at datetime

5. DispatchItemCode Master (dispatch_item_codes)

  • id bigint (PK)
  • product_id bigint (FK -> products.id)
  • uid varchar (Index -> inward_item_codes.uid)
  • quantity int (Default -1)
  • status varchar (Sold / Returned)
  • updated_by varchar
  • created_at / updated_at datetime

6. User Master (users)

  • id bigint (PK)
  • name varchar (Full Name)
  • email varchar (UserID, Unique)
  • password varchar (Hashed)
  • status varchar (Active / Inactive)
  • created_at / updated_at datetime

Terminal Commands Cheatsheet

1. Running the System Locally

Run these commands in separate terminal tabs in your project folder:

# Start PHP backend server
php artisan serve
# Compile styles in real-time
npm run dev

2. Database Migration Commands

# Run pending migrations
php artisan migrate
# WIPE database and rebuild tables fresh
php artisan migrate:fresh

3. Creating Seeders (Dummy Data)

# Run all seeders (populates default values)
php artisan db:seed
